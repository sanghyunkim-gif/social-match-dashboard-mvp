# Design Upgrade Brief

## 현재 상태
- **프로젝트**: PLAB Social Match Dashboard (KEVIN)
- **프레임워크**: Next.js 14.2.5 (App Router) + TypeScript
- **CSS 방식**: 순수 CSS + CSS 변수 (40개 토큰)
- **UI 라이브러리**: 없음 (커스텀)
- **폰트**: IBM Plex Sans KR (본문), Sora (디스플레이)
- **디자인 품질 점수**: 78/100
- **화면 수**: 2개 (로그인, 대시보드)

## CBO 컨텍스트

### 타겟 유저
- PLAB 내부 운영팀 (매치 데이터 분석)
- 경영진/리더십 (KPI 의사결정)
- 기술 숙련도: 보통~높음
- 데이터 리터러시: 높음

### 경쟁사 UI
- 내부 도구이므로 직접 경쟁사 없음
- 유사 도구: Amplitude, Mixpanel, Looker 등 분석 대시보드

### 포지셔닝
- "PLAB 풋살 매치 운영 효율을 데이터로 보는 유일한 내부 도구"
- 빠른 의사결정을 돕는 데이터 중심 인터페이스

### 가치 제안
- 복잡한 매치 운영 데이터를 한눈에 파악
- AI 기반 자연어 질의로 비전문가도 데이터 접근 가능

## 브랜드 방향

### 키워드
- 깔끔하고 전문적인 (Professional)
- 고급스럽고 세련된 (Premium)
- 신뢰감 (Trustworthy)
- 데이터 중심 (Data-driven)
- 효율적 (Efficient)

### 레퍼런스
- Linear (미니멀 + 빠른 인터랙션)
- Vercel Dashboard (개발자 중심 깔끔한 대시보드)
- Amplitude (분석 대시보드 전문성)

### 안티 레퍼런스
- 이모지 아이콘 남발
- 과한 그라데이션/네온 색상
- 기본 Tailwind 파란색

## 요청 범위
- **score 78 → "디자인 시스템 정교화 + 프리미엄 폴리싱"**
- 기존 CSS 변수 시스템을 더 세련되게 업그레이드
- 색상 팔레트 정교화 (프리미엄 느낌)
- 컴포넌트 폴리싱 (그림자, 테두리, 간격)
- 이모지 아이콘 교체
- 시각적 위계 강화

## UI 라이브러리 결정
- 현재: 없음 (순수 CSS)
- 유지: 순수 CSS + CSS 변수 (기존 시스템이 잘 구축되어 있으므로)
